## 1. Mission, Audience, Voice

### Mission

Помочь человеку **услышать себя** в мире, который непрерывно подсказывает ему чужие цели. Сделать практику самоопознания, которая опирается на собственный метод автора и не требует постоянного присутствия коуча.

Это не приложение для расслабления, не todo-приложение, не дневник эмоций. Это **карта для размышления о себе** с конкретной структурой (4 слоя), доступная для возврата.

### Target audience

Holiday-юзеры на старте — **русскоязычная аудитория 25-40 лет**, склонная к рефлексии, осознающая, что "живёт чужими целями". Через 6-12 месяцев — английский рынок аналогичной аудитории.

**Три ключевые боли, которые решаем:**
1. *"Я живу чужими целями и не понимаю, чего хочу сам"* — главная, отличает нас от конкурентов
2. *"Ставлю цели, но не достигаю/бросаю"* — связь цель↔ресурс↔опора
3. *"Внутри тревога, шатает, нет опоры"* — корни как точка возврата

Боль №1 — главный месседж для лендинга и сторов. Она массовая и слабо закрыта существующими продуктами.

### Voice

- **"Ты"**, не "вы" — теплее, близко, как разговор с собой
- **Спокойный, не назидательный** — продукт уважает темп юзера
- **Конкретный, без эзотерики** — "корень" объяснён через функцию ("точка возврата"), а не через "энергию"
- **Допускает паузу и тишину** — не давит уведомлениями, не геймифицирует инсайты
- **Признаёт что не идеально** — "раскрывается полнее когда вдумчиво", а не "работает только если"

### Author

**Вячеслав** — коуч по самоопоре. Создатель метода и продукта. Имеет Telegram-канал @snowvb. В продукте присутствует в одном месте — блок "От автора" в конце страницы "О подходе".

Личный бренд встроен в продукт намеренно, но мягко: дерево несёт **метод**, не **личность**. Если юзер захочет к автору лично — найдёт через Telegram. Если нет — продукт работает самостоятельно.

## 2. Product Principles (Non-Negotiables)

Это правила, которые НЕ нарушаем без явного решения и обновления документа.

1. **Privacy first.** Содержимое дерева глубоко личное. Никогда не индексируется, не шарится без явного действия юзера, не используется в обучающих данных без opt-in.
2. **No addiction patterns.** Не daily-streak-приложение. Не геймифицируем инсайты ("ты на 73% к озарению!"). Не оптимизируем session duration в плюс.
3. **No popups, no upsells in the middle of work.** Pop-ups только на logical break-points (после первого плода, после возврата). Никогда — во время работы с деревом.
4. **Respect emotional weight.** Это инструмент рефлексии. Анимации, тосты, копирайт — всё спокойное. Никаких "Wow! Great job!" Никаких эмодзи в систематических сообщениях.
5. **The user's text is sacred.** Никогда не модифицируем, не "улучшаем" формулировки юзера. Орфография — его, не наша.
6. **Mobile is a first-class citizen** (в Production, не в прототипе) — будут пользоваться с телефона на ходу или дома на диване.
7. **Russian first, English second.** До поиска PMF — только русский. Архитектурно готовы к i18n с первого дня.

## 3. User Journey

### CJM — первый визит

1. Открывает (через ссылку, рекламу, шеринг друга)
2. **Welcome-модалка**: слоган + введение (~90 слов) + кнопка "Понятно, начнём"
3. Видит **пустое дерево** + правую панель с 4 кнопками "Добавить"
4. Кликает `i` рядом с любой кнопкой → **popover с определением** + ссылка "Показать пример"
5. Опционально открывает пример → видит **modal с примером** ("Корень 'шапка'", "Ресурс 'терпение'" и т.д.)
6. **Заполняет любую из зон** — порядок не навязываем
7. При **первом плоде** → **диалог "На какой срок ты сейчас смотришь?"** (6m / 1y / 3y, можно пропустить)
8. Открывает **левую шторку с вопросами** для рефлексии — табы по зонам, 3 уровня глубины
9. Опционально читает **"О подходе"** (через подвал или меню)

### CJM — возврат

1. Открывает, видит дерево в том состоянии в каком оставил
2. **Welcome НЕ показывается** (флаг `tree_intro_seen_v1` в localStorage)
3. Может **отметить плоды как "сорванные"** через клик → меню → "Сорвать плод"
4. Может **поменять период реализации** через меню или pill-кнопку в шапке
5. Может **перетащить любой элемент** мышкой (опоры, ресурсы, плоды)

### CJM — конец сессии

Юзер закрывает вкладку. Все данные в localStorage. При следующем открытии — продолжит с того же места.

В Production-версии: данные синхронизированы с Supabase, доступны с разных устройств после auth.

## 4. Data Model

### Entities

```typescript
// Корень / опора
interface Root {
  id: string;          // uuid
  text: string;
  slotId?: string;     // 'root-1' .. 'root-6' — если из стандартного слота
  x?: number;          // 0-100, если перетащен
  y?: number;
  createdAt: timestamp;
  updatedAt: timestamp;
}

// Ресурс / ствол
interface TrunkItem {
  id: string;
  text: string;
  x?: number;          // 0-100, если pinned
  y?: number;
  pinned: boolean;     // true = юзер перетащил, не пересчитывать автопозицию
  createdAt: timestamp;
  updatedAt: timestamp;
}

// Ветка фокуса
interface Branch {
  id: string;
  text: string;
  slotId: string;      // 'branch-1' .. 'branch-5' (обязательно)
  createdAt: timestamp;
  updatedAt: timestamp;
}

// Плод
interface Fruit {
  id: string;
  text: string;
  x: number;           // 0-100 (всегда заданы)
  y: number;
  harvested: boolean;  // отмечен как достигнутый
  branchId?: string;   // в текущей версии НЕ привязан жёстко, перетащим со временем
  harvestedAt?: timestamp;
  createdAt: timestamp;
  updatedAt: timestamp;
}

// Дерево как контейнер
interface Tree {
  id: string;
  userId: string;
  name: string;        // 'Моё дерево' по умолчанию
  horizon?: '6m' | '1y' | '3y';
  roots: Root[];
  trunk: TrunkItem[];
  branches: Branch[];  // макс 5
  crown: Fruit[];
  createdAt: timestamp;
  updatedAt: timestamp;
}

// Юзер
interface User {
  id: string;
  email?: string;       // null для анонимов
  tier: 'free' | 'premium';
  subscriptionStatus?: 'active' | 'past_due' | 'canceled';
  language: 'ru' | 'en';
  createdAt: timestamp;
  lastActiveAt: timestamp;
}
```

### Constraints

- Roots: до 6 в slotах; больше — спавнятся возле земли (y=88) с offset
- Branches: **жёсткий лимит 5**, попытка добавить 6-ю → toast "Сейчас 5 из 5..."
- Trunk: без лимита, новые спавнятся в центр круга с лёгкой спиралью
- Fruit: без лимита, спавнятся на свободных позициях апельсинов из `ORANGE_POSITIONS`

### Storage

**Прототип:** localStorage с ключом `treeItems_v6`, `tree_intro_seen_v1`, `tree_horizon_v1`.

**Production:** Supabase Postgres. Auth через Supabase. Анонимные сессии (до регистрации) хранятся в localStorage, при регистрации мигрируем в БД.

### 7.3 Zone Definitions (popover при клике `i`)

#### Корни — Опоры (твоя точка возврата)

> То, что возвращает тебе ощущение опоры и тепла, когда внутри тревога, упадок или сомнение. Люди, события, ценности, символы — всё, что напоминает: «я уже справлялся, у меня есть на что опереться». Корни не решают проблему — они возвращают спокойствие, из которого можно действовать дальше.

#### Ствол — Ресурсы (то, что у тебя есть сейчас)

> Реальные ресурсы, которые работают на тебя последние 3 месяца и будут доступны ещё 3 месяца вперёд. Деньги, навыки, время, отношения, опыт. Не «как должно быть», а как есть — включая то, что хотелось бы изменить. Ствол даёт чувство собственной ценности и делает путь к плодам видимым.

#### Ветки фокуса (куда ты направляешь энергию)

> Сферы, в которых ты хочешь расти прямо сейчас: спорт, отношения, бизнес, семья, обучение. У человека ограничено внимание — поэтому веток немного, до пяти. Ветка — это не цель. Это направление фокуса, в котором будут расти твои плоды.

#### Плоды (конкретные шаги)

> То, что ты собираешься сорвать. Не «хочу больше денег», а «200к в месяц к декабрю» — это главный плод. И промежуточные — как шаги, без которых невозможно сорвать главный. Чем конкретнее формулировка — тем выше вероятность успеха и вкус плода, даже если остановишься на промежуточном.

### 7.4 Examples (в modals при клике "Показать пример")

#### Пример: Корень «шапка» или «семья»

> Корнем может быть как что-то конкретное (предмет, человек, событие), так и абстрактное — семья, юмор, дружба, свобода. Конкретика не важна. Важно чувство, которое возникает, когда смотришь на этот корень.
>
> Слово «шапка» у разных людей значит разное:
> — У одного — событие: ребёнком увидел шапку и так впечатлился, что теперь сам делает шапки.
> — У другого — человек: ассоциация с близким, при взгляде встаёт его образ.
> — У третьего — символ: ощущение себя в моменте, когда надевал шапку, например любовь к маме.
>
> Главное не само слово, а отклик внутри.

#### Пример: Ресурс «терпение» или текущий доход

> Одно слово — разная функция. «Терпение»:
> — Один записал в смысле «не спешить, дать процессу время». Когда видит — успокаивается.
> — Другой — как напоминание: раньше терпел во вред себе, теперь учится не молчать.
>
> Текущий доход тоже звучит по-разному:
> — Для одного это средство: возможность вложиться в курс, переезд, плод.
> — Для другого — достижение: видимый результат пути.
>
> Когда ствол заполнен — должно быть ясно видно, как именно эти ресурсы помогут сорвать каждый плод.

#### Пример: Ветка «Бизнес» — две версии

> Одна и та же ветка, разные намерения:
> — У одного «Бизнес» — почитать про рынок и подумать, в какую нишу заходить.
> — У другого «Бизнес» — найти первого клиента и закрыть сделку до конца месяца.
>
> Плоды отражают основное намерение развития ветки — то, куда ты тратишь свой драгоценный фокус, время и ресурсы. Поэтому название ветки лучше делать конкретным: не «Здоровье вообще», а «Восстановление сна» или «Силовые тренировки».

#### Пример: Плод — формулировка решает

> «Хочу больше денег» → мозг не понимает, что делать. Цель размыта.
> «200к в месяц к декабрю, чтобы переехать» → есть срок, сумма, смысл.
>
> Если ветка «Смена профессии», мастер-плод может быть «устроиться на новую работу к июню». Промежуточные:
> — Изучить рынок (и заранее решить, как поймёшь, что плод сорван).
> — Поговорить с человеком из этой профессии.
> — Пройти курс по нужному навыку.
>
> Так большая цель разбивается на достижимые шаги.

### 7.5 Questions for Reflection (левая шторка)

Структура: 3 уровня глубины в каждой зоне.

#### Опоры

**С чего начать:**
- Без чего ты не представляешь свою жизнь?
- Что или кто всегда был тебе опорой?

**Чтобы копнуть глубже:**
- На что ты опираешься столько, сколько себя помнишь?
- Что поддерживало тебя в самые трудные моменты — и помогает до сих пор?
- Кто или что было рядом в моменты, когда ты стал сильнее?

**Тело подсказывает:**
- Когда ты думаешь о том, что тебя по-настоящему держит на плаву — где в теле появляется тепло, спокойствие, единение? Если отклика нет — возможно, эта опора не твоя или больше не актуальна.

#### Ресурсы

**С чего начать:**
- Какие свои качества, достижения считаешь поводом для гордости?

**Чтобы копнуть глубже:**
- Какие ресурсы помогут тебе продвигаться ближе к вкушению плодов по каждой из веток?
- Какие ресурсы сейчас актуальны и будут ещё как минимум 3 месяца?

**Тело подсказывает:**
- При взгляде на ресурсы — какие вызывают ощущение «я могу», а какие напряжение или сжатие? Это сигнал: одни поддержать и усилить, к другим изменить отношение, увидеть силу и принять.

#### Ветки фокуса

**С чего начать:**
- Если бы у тебя была одна сфера, на которой нужно сосредоточиться полгода — какая?

**Чтобы копнуть глубже:**

- Какие ветки сейчас требуют твоего участия больше всего — куда хочешь направить ресурсы?

**Тело подсказывает:**
- Какие основные векторы в твоей жизни самые важные — прочти их название и следи за откликом тела, где замечаешь отклик? Если отклика нет — возможно, желание не лично твоё, а взято извне.

#### Раздел: Зачем дерево

> Нас не учат слышать себя. Мы живём по требованиям и ожиданиям мира — и постепенно учимся игнорировать собственные сигналы ради внешнего. Это дерево помогает увидеть себя с разных углов и придумать, как проявить это понимание в реальной жизни.

Реальный путь может отличаться — но твой главный инструмент, мозг, теперь знает твои настоящие желания. На пути ты сможешь опираться на эти мысли и чувства, когда станет сложно или страшно.

#### Раздел: Как устроено дерево

> *(в выделенном блоке terracotta):* Крона не вырастает шире, чем позволяют корни. Ствол не может включать ресурсы, выходящие за пределы того, каким ты себя ощущаешь. Корни не питают конструкцию, если ты не видишь, как опираешься на них в разных ситуациях. Всё связано.

> **Корни** — то, на что ты опирался, опираешься и будешь опираться. Родители, друзья, опыт боли, после которой ты стал сильнее. В домиках внизу дерева ты собираешь эти опоры, видишь, как они помогали, замечаешь связь с твоими мечтами.

> **Ствол** держится прочно за счёт корней. Это твои актуальные ресурсы: победы, поражения, работа, деньги, качества, которые укрепляют опору на себя — те, которыми гордишься и открыто проявляешь, и те, что прячешь или хочешь трансформировать. Ствол — это видимая, честная картина «что у меня сейчас есть, без деления на хорошо или плохо, просто как есть, без оценок».

> **Крона** растёт настолько, насколько ты ухаживаешь за корнями и направляешь свет внимания на дерево. Корни уходят глубже, когда ты замечаешь моменты опоры. Крона расширяется, когда ты признаёшь свои ресурсы. Плоды зреют, когда ты понимаешь: «у меня есть или будет время, желание и ресурс для этого».

#### Раздел: На каких подходах основано

**Карточка 1: ACT — Acceptance & Commitment Therapy**
> Терапия третьей волны: принятие, осознанность, ясность ценностей, действие из ценностей. Корни ↔ ценности, Ствол ↔ принятие реальности, Ветки и Плоды ↔ committed action. Мета-анализ 65 исследований (Springer, 2025) показал умеренный эффект ACT (Hedges' g = 0.72) на снижение тревоги и депрессии и рост психологической гибкости.

**Карточка 2: Implementation Intentions**
> Формулировка целей в формате «когда X, то Y» удваивает шанс их достижения. Мета-анализ 94 исследований Голвитцера и Шерана показал эффект d = 0.65 (medium-to-large). Поэтому плоды формулируются конкретно — с местом, временем и ресурсом.

**Карточка 3: Self-compassion (Кристин Нефф)**
> Заполнять дерево нужно с теплом к себе, без самокритики. Мета-анализ 56 RCT (Mindfulness, 2023) подтвердил, что self-compassion интервенции снижают депрессию, тревогу и стресс. Корни — это инструмент самосострадания: «у меня есть на что опереться».

**Карточка 4: Интроспекция и интероцепция — взгляд внутрь**
> Дерево раскрывается полнее, когда заполняешь его, заглядывая внутрь, а не на автомате. Интроспекция — это наблюдение за своими мыслями, ценностями и чувствами; она отвечает на вопрос «что я думаю и чувствую». Интероцепция — способность слышать сигналы тела: «где это живёт и есть ли настоящий отклик». Вместе они защищают от того, чтобы заполнить дерево чужими, заученными ответами. Систематический обзор (Frontiers, 2020) показал: чем выше осознанность тела, тем эффективнее регуляция эмоций. Поэтому в каждой зоне есть вопрос «что говорит тело».

#### Раздел: Дерево живое

> Дерево не делается один раз и навсегда. Реальность меняется — у тебя появляются новые ресурсы, что-то отпадает, фокус смещается. Возвращайся к дереву, когда чувствуешь, что внутри что-то сдвинулось.

>
> Это не отчётность. Это способ оставаться в контакте с собой по мере того, как ты меняешься. Дерево, которое отражает реальность, помогает. Дерево-галочка, которое стоит без движения — нет.

## 8. Interactive Behaviors

### 8.1 Drag & Drop

Все элементы Roots / Trunk / Crown — draggable. Branches — НЕ draggable (фиксированные слоты).

Логика:
- `mousedown` на элементе → флаг "dragging" с порогом 3px движения
- Если меньше 3px движения → considered click (open menu / edit)
- При drag: позиция считается в процентах от containerRef (ref на div-обёртку картинки)
- При drop: x, y сохраняются; для Trunk выставляется `pinned: true` (auto-positioning больше не действует)

### 8.2 Harvest mechanic

Клик по плоду → mini-menu (Редактировать / Сорвать / Удалить).

При "Сорвать":
- `harvested: true` записывается
- Плод визуально меняется: цвет → приглушённый коричневый, текст зачёркнут, появляется зелёная галочка ✓, **CSS-маска** добавляет надкус справа-сверху (через `mask-image: radial-gradient`)

Счётчик в правой панели: `✓2 · 5` — сорвано 2 из 5 всего.

### 8.3 Welcome flow

- При первом open приложения (нет `tree_intro_seen_v1` в localStorage) → показывается Welcome modal
- Закрытие → флаг ставится → больше не показывается
- Повторный показ — через меню "Снова показать введение"

### 8.4 Horizon flow

- При первой попытке добавить плод (и если `tree_horizon_v1` не установлен) → диалог горизонта
- После выбора (или пропуска) → продолжается добавление плода
- Pill в шапке правой панели показывает выбранный горизонт
- Клик по pill или меню → можно поменять

### 8.5 Questions drawer

- По умолчанию закрыта
- Вертикальный trigger слева ("Вопросы для рефлексии") — клик открывает
- Внутри: табы (Опоры / Ресурсы / Ветки фокуса / Плоды) + 3 раскрывающиеся секции
- Закрытие — крестик в шапке или клик по фону

### 8.6 Info popovers (`i` rядом с кнопками)

- Клик `i` → popover снизу с определением зоны + ссылка "Показать пример"

### Принципы архитектуры (Architecture Principles)

1. **Server-first где можно** — данные через Server Components, мутации через Server Actions. Client Components только там, где нужна интерактивность (drag, popovers).
2. **Feature flags везде** — каждая премиум-фича закрыта `featureFlags.has('multi-tree')`. По умолчанию false. Можно включить per-user через PostHog flags без релиза кода.
3. **`isPremium` поле в user model с первого дня** — даже если все пока free, инфраструктура готова к paywall.
4. **No localStorage для critical data в Production** — всё через Supabase. localStorage только для анонимной сессии до auth.
5. **Все CDN-зависимости запинены** — version locks в package.json + lockfile. Не "latest", не `^` без необходимости.
6. **Шрифты — через next/font/google** — оптимизированная загрузка, без runtime сюрпризов от Google Fonts API.
7. **Никакого in-browser компилятора** — JSX компилируется на build-time через SWC (встроен в Next.js).
8. **No console.log в Production** — через ESLint правило. Используем PostHog/Sentry для логирования.
9. **Тесты:** Unit-тесты для бизнес-логики (drag bounds, harvest toggle, slot allocation), E2E через Playwright для критичных flow (welcome → first root → first fruit + horizon).
10. **CI/CD:** GitHub Actions → Vercel preview deploy on PR, auto-deploy main to production.

## 10. North Star Metric

### Метрика

**Weekly Active Returners (WAR)** — % пользователей, которые **вернулись хотя бы один раз** в течение недели после первого визита.

### Почему именно эта метрика

1. **Wellness-продукт ≠ daily-продукт.** Большинство bullet metrics не подходят (DAU, MAU мерят не то). Дерево предполагает рефлексивные сессии, а не ежедневное использование. Возвращение через неделю — здоровый сигнал ценности.
2. **Возврат = leading indicator PMF.** Если люди возвращаются — у продукта product-market fit, можно строить дальше. Если нет — никакая оптимизация воронки не спасёт, надо менять core.
3. **Анти-метрика к токсичным паттернам.** Не толкает к streaks, push-notifications, dark patterns. Поощряет органическое возвращение.
4. **Совместима с бизнес-моделью.** Возвращающиеся — основа для подписки. Не возвращаются — никакая подписка не сработает.

### Условия смены North Star

Менять основную метрику = серьёзный продуктовый pivot, требует решения совместно (Vyacheslav + Claude как продакт). Триггеры рассмотрения смены:

1. **Если WAR стабильно >40% на 3-месячном горизонте** — продукт явно работает, новая бочка — глубина engagement. Возможна смена на **"Trees Completed"** (полное заполнение) или **"Fruits Harvested per User"**.
2. **Если WAR стабильно <10% несмотря на оптимизации воронки** — продукт не цепляет. Прежде чем менять метрику — нужно понять почему (интервью). Если онбординг чинится — остаёмся. Если ценность не доходит — pivot продукта, не метрики.
3. **Если запускаем premium с conversion > 5%** — добавляем вторичную метрику **MRR**, но WAR остаётся главной leading-метрикой engagement.

### Supporting metrics (всегда мерим)

- **D7 / D30 / D90 retention** разрезанные по точке "содержательного заполнения"
- **Conversion в "первый момент ценности"** (заполнил минимум по 1 элементу в каждой зоне)
- **Conversion в "содержательное заполнение"** (≥3 корня, ≥3 ресурса, ≥2 ветки, ≥3 плода)
- **Conversion в "завершённое дерево"** (≥5/≥5/3-5/≥5)
- **Fruit harvest rate** (% юзеров с ≥1 harvested fruit)
- **About page open rate** (proxy для engagement)
- **Coach link clicks** (proxy для перехода в личную практику)

### Anti-metrics (НЕ оптимизируем вверх)

- **Session duration** — мерим как diagnostic, не как target
- **Number of edits per session** — мерим, но не давим в сторону "больше"
- **Notification engagement** — у нас почти нет уведомлений, и не хотим обратного

## 15. Research Base

### Размер рынка

- Mental wellness apps: **$6-9B в 2024-2025**, CAGR ~15-17%, прогноз $22-45B к 2033-2035 (разные источники сходятся на двузначном росте)
- Subsegment "wellness management" растёт быстрее всех — ~17.91% CAGR
- Self-development / introspection — узкая ниша внутри, недо-обслуженная сегодня

### Competitive landscape

| Игрок | Фокус | Где мы отличаемся |
|------|-------|------------------|
| Calm | Расслабление, медитации | Мы про самопознание и цели, не про "успокоиться" |
| Headspace | Mindfulness + sleep | Аналогично — другая боль |
| BetterHelp / Talkspace | Терапия с живым человеком | Мы — практика без человека (но с авторским методом) |
| Stoic, Reflectly | Daily journaling | Мы — структурная карта, а не daily writing |
| Finch | Care + cute pet | Мы — взрослый рефлексивный инструмент |
| 16personalities, MBTI | Personality test | Мы — не тип, а контекст моей жизни |
| Best Self Journal | Goal-setting структурой | Мы цифровые + связываем цели с опорами/ресурсами (они не делают) |

**Наша позиция:** в пересечении "self-knowledge" + "goal-setting" + "live document" — никто не закрывает все три одновременно.

### Психологическая база (см. About page для full text)

1. **ACT** — Hayes, Harris. Meta-analysis Hedges' g = 0.72 на тревогу/депрессию (Springer 2025).
2. **Implementation Intentions** — Gollwitzer & Sheeran. d = 0.65, meta of 94 studies.
3. **Self-compassion** — Kristin Neff. Meta of 56 RCTs (Mindfulness, 2023).
4. **Interoception** — Damasio, Garfinkel. Systematic review (Frontiers, 2020).
5. **Polyvagal theory practice** — Deb Dana, *Anchored: How to Befriend Your Nervous System*.

### Почему мы верим в монетизацию

- Wellness sector — один из самых монетизируемых в B2C SaaS
- Conversion rate freemium → premium в нише: **2.18% median** (Lenny's Newsletter / RevenueCat data)
- ARPPU в нише: $30-60/год
- При WAR > 25% и 1000 активных юзеров: ~$600-1200/мес revenue at minimal scale, scaling to ~$50-300K/year by year 2-3 with organic growth + light paid marketing
- **Защита от конкурентов:** уникальный метод + личный бренд автора → defensible, нельзя скопировать "просто скопировав фичи"

### Почему людям полезно (психологические "точки удара")

1. **Externalization of self.** Видеть себя со стороны, не варясь внутри. Психологически doku мощно (gestalt, schema therapy).
2. **Reduces cognitive load.** Удерживать в голове 6 опор + 5 ресурсов + 5 веток + 5 плодов одновременно невозможно. Дерево их *удерживает за тебя*.
3. **Connects past–present–future.** Корни (прошлое) ↔ Ствол (настоящее) ↔ Крона (будущее). Большинство приложений работают только с одной из этих временных плоскостей.
4. **Embodied reflection.** Вопросы про тело — редкость в digital-продуктах. Это даёт unique experience и реально работает (по интероцепции исследования).
5. **No pressure.** Не daily-streak, не "проиграл если пропустил". Уважение к темпу = меньше churn по моральным причинам.

### За счёт чего возьмём долю рынка

1. **Уникальный метод "самоопора"** — нет аналогов в текущих продуктах
2. **Личный бренд Вячеслава** — autoritet и trust из коуч-практики
3. **Виральность через share card** — экспоненциальный органик-рост возможен
4. **Defensible content** — vопросы, примеры, examples — авторский, не GPT-generated
5. **Нет конкурентов в точной нише** — пересечение трёх категорий уникально

## 16. Decision Log (Development Legends)

История архитектурных и продуктовых решений с rationale. Обновляется при каждом значимом решении.

### 2026-Q2 prototype phase

**D1. Использовали single-file HTML+React+Babel в браузере для прототипа.**
- Rationale: Скорость итерации. Минимум overhead для валидации идеи.
- Trade-off: Хрупкость к изменениям CDN-зависимостей (см. L1 в Lessons).
- В Production: уходим в Next.js.

**D2. Координаты слотов жёстко закодированы в процентах.**
- Rationale: Картинка дерева фиксированная. Проценты дают resolution-independence.
- Альтернатива: SVG-карта с координатами — overkill для текущего scope.
- Менять при: смене картинки дерева.

**D3. Лимит 5 веток фокуса — жёсткий.**
- Rationale: Психологическая нагрузка, теория ограниченного внимания (Miller, 7±2). Больше веток = расфокус. Это product principle.
- При жалобах: НЕ увеличивать механически. Сначала понять что **на самом деле** хотят — возможно нужны multi-tree.

**D4. Лимит 6 корней — мягкий (можно перетаскивать).**
- Rationale: 6 домиков на картинке. Перетаскиванием можно нарушить, и это ок — корни не должны давить лимитом.
- Поведение: первые 6 идут в слоты, дальше — спавн возле земли.

**D5. Drag самописный, не библиотека.**
- Rationale: В прототипе нужен был полный контроль и простота.
- В Production: переходим на `@dnd-kit/core` — touch-friendly из коробки.

**D6. Welcome-модалка показывается ОДИН раз (флаг в localStorage).**
- Rationale: Не давить. Юзер прочитал — больше не нужно.
- Возврат: через меню `⋯` → "Снова показать введение".

**D7. Horizon dialog появляется ПЕРЕД первым плодом, не на старте.**
- Rationale: На старте юзер не знает зачем выбирать. Появление в контексте (когда уже что-то добавил, дошёл до плодов) даёт мотивацию ответить осмысленно.

**D8. Все элементы кроме веток — draggable.**
- Rationale: Свобода позиционирования = ownership. Branches фиксированы потому что они отражают **структуру дерева** (5 веток в кроне), не контент.

**D9. Tree image на фоне через `background-image: cover` + blur, главное изображение — `<img>` с object-fit container.**
- Rationale: Контент сцены не обрезается на разных экранах, фон закрывает "пустые" сине-серые поля по бокам.
- Альтернатива (отвергнута): растянуть `tree.png` через cover на основной → теряем точность позиционирования.

**D10. Цветовая палитра — приглушённые землистые тона.**
- Rationale: Wellness-контекст. Яркие crayola-цвета (initial version) выглядели как "детский продукт", не как взрослый инструмент рефлексии. См. L2.

**D11. Шрифты Manrope (UI) + Fraunces (serif для бренда и заголовков).**
- Rationale: Pair используется в современных wellness/coaching-продуктах (близкие vibes Oura, Calm). Manrope — тёплый geometric sans, читаемый. Fraunces — editorial serif, добавляет авторитет без формальности.

**D12. Деревьев пока одно на юзера, multi-tree — premium-фича.**
- Rationale: Упрощает onboarding (нет выбора "какое дерево создаём"). Multi-tree приходит когда юзер уже привязался.

**D13. "Сорванный плод" визуально = надкус (CSS-маска) + зачёркивание + цвет приглушённый + галочка.**
- Rationale: Метафора прямая (сорвал — съел). Дофамин маленький, но осязаемый. См. также раздел про harvested_count как teaser виральности ("я сорвал N плодов").
- Реализация надкуса через `mask-image: radial-gradient` — надёжно работает во всех современных браузерах. Старая попытка через `clip-path: path()` была хрупкой.

**D14. North Star = Weekly Active Returners.**
- Rationale: см. раздел 10. Совместима с продуктом (не daily), читается как PMF-сигнал, защищает от dark patterns.

**D15. Аналитика — PostHog.**
- Rationale: см. сравнение в Tech Stack. Один инструмент для events + replays + flags + funnels.

**D16. Free-only at launch, premium через 2-4 месяца.**
- Rationale: см. раздел 13. Предотвращает преждевременную монетизацию и потерю виральности.

**D17. От автора — Вячеслав, "коуч по самоопоре".**
- Rationale: Уникальное позиционирование, прямая связь с метафорой продукта, отличает от широкой массы коучей. Trade-off: новое слово для холодного юзера, но в конце "О подходе" уже принято.

### 2026-07-17 Phase 0 → Phase 1 transition

**D18. Next.js 16 + Tailwind CSS v4 вместо заявленных "14+".**
- Rationale: На момент scaffolding `create-next-app@latest` ставит 16.2.10 / Tailwind v4 — новее, чем версия, названная в разделе 9, но удовлетворяет "14+". App Router, `next/font`, Server Components — всё как задумано.
- Заметка: Next 16 несёт breaking changes относительно того, что могло быть в обучающих данных модели — при генерации кода сверяться с `node_modules/next/dist/docs/` (см. авто-сгенерированный `AGENTS.md` в корне репо).

**D19. GitHub push через SSH-ключ, не через `gh` CLI.**
- Rationale: На машине фаундера нет Homebrew, `gh` CLI недоступен без него. Вместо установки Homebrew ради одного инструмента — сгенерирован новый ed25519 SSH-ключ, добавлен фаундером в GitHub-аккаунт вручную. Репозиторий `SnowVB/rootsnfroots` фаундер создал сам через веб-интерфейс.
- Trade-off: без `gh` CLI создание issues/PR из терминала потребует либо веб-интерфейса, либо установки `gh` позже (например через прямой бинарник с GitHub Releases, без sudo).

**D20. Vercel и Supabase подключаются через дашборд, не CLI.**
- Rationale: Оба сервиса требуют OAuth-логина фаундера в браузере — Claude Code не может пройти этот флоу от его имени. Vercel: `vercel.com/new` → import `SnowVB/rootsnfroots`. Supabase: проект создаётся в дашборде, ключи (`NEXT_PUBLIC_SUPABASE_URL`, `NEXT_PUBLIC_SUPABASE_ANON_KEY`) передаются в `.env.local` (не коммитится, см. `.env.local.example`).
- Реализовано заранее: `@supabase/ssr` + `@supabase/supabase-js` установлены, клиенты-хелперы в `src/lib/supabase/client.ts` (browser) и `src/lib/supabase/server.ts` (server, cookie-based session). Auth-flow и схема БД — отдельный шаг Phase 1, ждут реального Supabase-проекта.

## 18. Lessons Learned (from prototype)

### L1. In-browser Babel — хрупкая зависимость

- **Что произошло:** Новые версии `@babel/standalone` начали вставлять `import` statements в скомпилированный JSX (новый "automatic" JSX runtime). Safari отказывалась парсить это без module-context. Прототип ломался.
- **Решение в прототипе:** Запинили версию `7.24.7` + явно указали `data-presets="react"`.
- **В Production:** Никакого in-browser компилятора. JSX компилируется на build через Next.js (SWC).

### L2. Google Fonts API меняется

- **Что произошло:** URL с `Fraunces:opsz,wght@9..144,400;500;600` начал возвращать 400. Скорее всего временный сбой, но напомнил о хрупкости.
- **Решение в прототипе:** Убрали `opsz` axis, оставили только wght.
- **В Production:** `next/font/google` — оптимизированная локальная подгрузка, без runtime сюрпризов.

### L3. Картинка через `file://` в Chrome перестала работать

- **Что произошло:** Обновление Chrome ужесточило CORS для локальных файлов. `<img src="tree.png">` через file:// перестал грузиться.
- **Решение:** Python local server (`python3 -m http.server`).
- **В Production:** Картинки идут через Next.js Image Optimization, на CDN.

### L4. Цветовая палитра эволюционировала

- **Сначала:** Яркие "crayola" тона (saddleBrown #8B4513, forestGreen #228B22, ярко-зелёный, томатный оранжевый)
- **Стало:** Приглушённые землистые (терракота #B8896B, мох #7A8D6A, sage #AED581, коралл #E8876B)
- **Урок:** В wellness-продуктах сатурация важна. Яркое = детское/нервное. Приглушённое = взрослое/рефлексивное.

### L5. Drag UX требует careful state management

- Click vs drag conflict: пришлось вводить threshold 3px движения чтобы различать
- DeleteBtn внутри draggable требовал stopPropagation на mousedown, иначе родитель открывал edit
- InfoPopover на ту же кнопку требовал anchorRef чтобы корректно toggle'иться

### L6. Welcome-модалка эволюционировала через тестирование

- Старая: одно предложение про "не знают чего хотят"
- Новая: 90 слов с конкретикой + якорь про крону/корни/ствол + про возвращение
- Урок: первый экран нужен **достаточно длинный** чтобы поставить контекст, **не настолько длинный** чтобы отпугнуть.

### L7. Микро-копирайт делает продукт

- "Горизонт" → "Период реализации" — звучит конкретнее
- "Заполняй медленно" → "Заполняй медленно, вдумчиво" — два слова делают работу за абзац
- Слово "наследована" → "возможно желание не лично твоё, а взято извне" — проще, точнее
- Урок: микро-итерации копирайта продолжаются до самого конца. Никогда не считать что "написано окончательно".

### L8. Структура вопросов 3 уровня — паттерн

- "С чего начать / Чтобы копнуть глубже / Тело подсказывает" — рабочий паттерн прогрессии
- Юзер сам выбирает глубину, не давим
- Тело-вопрос — uniqueness нашего метода

### L9. Adding "если отклика нет" — мощная мини-разрешение

- Pattern: в тело-вопросах добавлять *"Если отклика нет — возможно, это не твоё или не актуально"*
- Это **разрешает юзеру отпустить** то, что не работает. Сильное для корней и веток (там часто пишут "правильные" чужие опоры/желания).

### L10. Premium-архитектура с дня 1, активация позже

- Поле `tier: 'free' | 'premium'` в user model сразу
- Feature flags — сразу
- Активация — только после данных и valid PMF
- Урок: предотвращает rewrite. Стоимость "заложить" низкая, стоимость "переделать архитектуру" высокая.

### L11. North Star метрика должна быть не "feel good" а "predictive"

- DAU/MAU — feel good но обманчивы (могут расти на спам-фичах)
- Session duration — anti-pattern в wellness
- WAR (Weekly Active Returners) — predictive: если люди возвращаются, продукт работает

### L12. Каждое решение → в Decision Log

- Без него через 3 месяца не помнишь почему сделано так. Особенно с AI-партнёром, где сессии конечны.
- Decision log = страховка от перепеределывания одного и того же.
